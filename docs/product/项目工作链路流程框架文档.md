产品工作流程

一上来用户可能上传了文件资料。

设置一个resource pipeline（资料处理流水线）：“文件资料管理ai”

“文件资料管理 AI ”，拆成四个模块：

1. Resource Library（资料库）
负责接收用户上传的文件，保存原文件、文件 metadata（元数据）、解析结果、章节结构、文本片段和必要的索引。它只负责让资料变成可查、可定位、可引用的资源，不负责决定本轮是否使用资料。

2. Resource Indexer（资料索引器）
负责把上传资料拆成可检索的结构，例如章节、页码、标题路径、正文片段、图表或公式位置。它生成的是通用资料结构，不写入具体学科、教材或考试规则。

3. ResourceResolver（资料证据选择器）
当用户的学习或文档需求已经比较清楚后，ResourceResolver 根据本轮需求去资料库中检索相关内容，判断是否有可用证据。如果找到相关资料，它只返回命中的章节、位置、概览和置信度，不直接替用户决定必须使用资料。

4. Evidence Handoff（证据交接）
如果系统认为资料可能有帮助，就把命中的资料位置、章节概览和引用理由展示给用户，询问是否参考这些资料来生成或扩写 Board（右侧文档）。用户确认后，证据包才会交给 BoardEditor（文档编辑 AI）生成或修改文档；用户拒绝时，BoardEditor 只根据用户原始需求生成，不使用这批资料。

资料使用流程：
用户上传资料
-> Resource Library 保存原文件和 metadata
-> Resource Indexer 解析章节、片段和索引
-> 用户提出学习 / 练习 / 写作需求
-> 需求清晰后，ResourceResolver 检索相关资料
-> 如果没有相关资料：只根据用户需求继续生成或扩写文档
-> 如果有相关资料：向用户展示命中章节、位置和概览
-> 用户确认参考：形成 Evidence Bundle（证据包），交给 BoardEditor
-> 用户拒绝参考：本轮不注入资料证据
-> 生成、扩写或编辑后的文档记录资料来源线索

用户的学习需求清晰后，将用户的学习需求在用户生成的文件资料中继续检索，上传的文件资料中有没有和当前用户的学习需求相关内容？

如果有：将命中了的相关内容的位置章节和章节概览发给用户，问是否参照相关章节来生成板书？如果用户同意就参照相关章节来生成板书；如果否定就不参照相关章节来生成板书，只参考用户一开始阐述的的学习需求目的生成板书。

如果没有：参考用户一开始阐述的的学习需求目的生成板书。



链路判断1.：用户在聊天框中和chatbot聊天对话机器人是在闲聊天还是有学习工作需求

1.1无需求：
对话框机器人陪用户闲聊天，普通聊天不能更新 LearningRequirementSheet（学习需求清单），不能读取右侧全文摘要，不能误入板书任务链路。
1.2有需求：
到链路判断2.
1.3意图不清或者过宽：
用户想学的内容表达不清或者表达清楚后但主题过宽，在聊天对话框中为用户进行引导和内容推荐，比如进一步向用户询问探寻，给一些内容主题推荐，不直接生成



链路判断2.：文档的情况，其中是否有用户的需求内容？如果有的话在什么位置？

2.1板书就是空白的：
走链路判断3。
2.2.板书中能找到：TargetRange（目标文本范围）要对哪一段已有内容进行某些动作 ；或InsertionAnchor（插入锚点）新内容要插到哪个位置。
走判断4.
2.3 target_not_resolved（目标未解析）没找到用户指定的文档框中的内容，可能是板书中没有相关内容，或者可能是内容位置不清：
进一步向用户交互，探寻追问并且同时通过文中的文段内容的相关性猜测相关内容位置，同时推荐如果扩写的话那些位置最合适。
比如用户说“讲这里”“改第二段”，系统要定位到具体段落。多候选、找不到、定位不可靠时，要对用户追问进一步探寻更具体准确的目标位置，比如“是第一章的第二段吗？或者是哪部分第二段？”。 一直以探寻用户需求内容位置的动机进行交互。


链路判断3：是要学还是练习？

3.1学某知识点 细化到指定知识点后：
走首次从零生成文档链路。走文件资料管理ai判断，然后行“扩写动作”走链路5.为此生成知识点讲解内容文档内容。
并且生成后问用户是否对新生成的内容从头进行讲解，如果用户回答是那么开讲？如果用户同意那么开始以小节为单位为用户从头进行内容讲解。
3.2做某项相关练习 那么要填写相关“清单”后：
走文件资料管理ai判断，行“扩写动作”走链路5.为此生成练习内容文档。并且生成后问用户是否从头进行讲解，如果用户回答是那么开讲。
清单内容要有：“1.用户要练习什么内容”“2.用户当前领域的水平”“3.用户练习相关内容的目的是什么”。聊天对话框ai追问用户清单内容来完善清单，清单填写完善前ai不主动进行下一步工作，除非用户要求直接生成。



链路判断4:对指定的 TargetRange（目标文本范围）或InsertionAnchor（插入锚点）做什么？

聊天对话框的chatbot聊天对话ai根据用户透露的信息，填写任务清单，清单填写完善充分后将清单交给板书文档管理ai
清单中固定收集四项：
目标位置
动作类型
问题 / 主题
是否有特殊互动要求

4.1讲 对TargetRange（目标文本范围）在聊天对话框中进行讲解：
讲解路线必须先由 Board AI 生成 BoardExplanationDirective（板书讲解指令），
其中包含目标摘录、讲解边界和允许讲解状态。
Chatbot 只能根据这个 directive 讲解。
4.2写 对文档进行编写：
将编写需求进行拆分，把用户一句自然语言请求，拆成结构化的BoardMutationPlan（板书变更计划），生成“文档动作清单”：
- edit：改写已有内容
- write：新增内容
- delete：删除内容，属于高风险动作，需要确认

比如用户说：“把这一节改得更通俗一点，再补一个例子。”
要将这一整句话拆分成多个需求：
edit：改写已有内容
write：新增例子

然后将走链路判断5.执行所有需求
 4.3在指定规则下互动：
当用户表达自己有“特殊的交互方式要求”时，中用户要对特定位置的某指定部分内容TargetRange（目标文本范围）进行“聊天互动”，如果我最后选择的是聊的这条链路,那么就进入一个循环，根据用户的特殊的交互方式要求给AI的prompt（提示词）里面多定义一套规则,规则定义了ai跟用户之间该如何循环交互。而如果用户突然想要表达内容是规则之外的,那就跳出循环，回到最开始。
比如用户说：“我们来扮演这个情景对话的两个角色，我是客人你是服务员，我们轮流读课文读对话台词。”这就是一个对话规则，如果用户念了课文中的客人的对话内容，你就说下一句服务员的对话。具体如何判别用户说的内容是否在规则内，就把“合规”的用户输入先整理下来，比如先把客人的台词都截下来，或者定义出“。。。这么样的输入内容是合规的”如果用户说的在规则台词范围内，ai就也按照规则输出。
用户输入不合规时可以先 rule_violation（规则内纠错），不一定立刻退出；用户明确结束才 exit_rule；用户提出新任务才 new_task 回到总链路。然后重新将用户输入从头走链路判别，走之前设计好的链路



链路判断5 具体怎么写：

5.1 编辑改写已有内容：
系统根据 BoardMutationPlan（板书变更计划）修改TargetRange（目标文本范围）内容，并写入版本历史。
编辑完成后，Chatbot 只说明已完成修改，并询问用户是否需要讲解本次改动或新版本内容。
如果用户确认需要讲解，再进入 BoardExplanationDirective（板书讲解指令）链路
5.2扩写新增内容：
走文件资料管理ai判断 后，找到指定的InsertionAnchor（插入锚点）后进行“扩写动作”走链路6.


链路判断6：扩写的时候篇幅多大？

篇幅尺度分类：
句子
段落
小节
文章

6.1如果是从零开始生成板书：
篇幅尺度为以文章为单位的篇幅大小

6.2如果是已经有的板书：
属于当前文档主题 -> 在相关父章节下新增
是独立主题 -> 询问创建新 lesson 还是追加到当前文档
用户明确要完整新板书 -> whole_board（整篇板书级）

此产品没有总调度器
